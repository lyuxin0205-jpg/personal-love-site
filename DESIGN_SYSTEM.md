# DESIGN_SYSTEM

## 设计基调

当前设计系统服务于一个低饱和、日系、自然光感的情侣生活记录网站。关键词是：奶绿色、青绿色、浅蓝色、米白、胶片、夏日晚风、城市生活、真实照片。

不建议出现：

- 黑金、商务奢华、Web3、赛博风。
- 过强玻璃拟态。
- 过度规整的模板卡片。
- 装饰性图标堆叠。
- 大面积高饱和渐变。
- 英文模板感文案。

## 字体规范

当前字体来源：

- 全局 sans：`ui-sans-serif`, `-apple-system`, `BlinkMacSystemFont`, `SF Pro Display`, `Inter`, `PingFang SC`, `Microsoft YaHei`, `sans-serif`
- 标题 serif：`.cinema-title` 使用 `ui-serif`, `New York`, `Iowan Old Style`, `Times New Roman`, `Songti SC`, `serif`

使用原则：

- 中文优先，中文排版要自然。
- 正文使用 sans，保持干净和可读。
- 大标题、故事标题、数字可使用 `.cinema-title`。
- 不使用负字距。当前全局 `letter-spacing: 0`。
- 避免标题过大压迫页面。Hero 当前使用 `clamp(2.35rem, 5.8vw, 5.6rem)`。
- 正文行高应宽松，当前常见 `leading-7`、`leading-8`、`leading-9`。

推荐字号：

- Hero 标题：`clamp(2.35rem, 5.8vw, 5.6rem)`
- Section 标题：移动端 `text-3xl`，桌面 `sm:text-5xl`
- 正文：`15px` 到 `19px`
- 说明/日期：`12px` 到 `14px`

## 配色规范

当前 CSS 变量定义在 `app/globals.css`：

| Token | 当前值 | 用途 |
| --- | --- | --- |
| `--ink` | `#f4f6eb` | 主背景，米白奶绿 |
| `--ink-soft` | `#e5efe6` | 柔和浅背景 |
| `--cream` | `#315f5a` | 主文字，深青灰/墨绿 |
| `--warm` | `#6f9f91` | 次级文字、温柔青绿 |
| `--gold` | `#94bc67` | 黄绿色点缀 |
| `--rose` | `#d68f7e` | 错误或少量暖色提示 |
| `--glass` | `rgba(255, 255, 248, 0.68)` | 轻玻璃背景 |

页面背景：

- 顶层为米白、浅绿、浅蓝纵向渐变。
- 可叠加很轻的颗粒和空气感。
- 不使用黑金，不使用强烈紫蓝渐变。

颜色使用原则：

- 主文字优先用深青灰/墨绿。
- 点缀色只用于按钮、进度、少量状态。
- 白色背景要带一点米色或透明度，避免纯白冷感。
- 阴影使用青灰或墨绿透明色，不使用黑色重阴影。

## 间距规范

当前页面间距来自 Tailwind：

- 页面横向：移动端 `px-5`，中屏 `sm:px-8`，桌面 `lg:px-14`。
- section 纵向：常见 `py-20`，生活碎片为 `py-14`。
- section 标题下方：`mb-10` 到 `sm:mb-12`。
- 卡片/纸张内部：`p-4`、`p-5`、`p-6`、`sm:p-7`、`sm:p-8`。
- 相册间距：columns `gap-5`，照片 `mb-6`。

使用原则：

- 保持呼吸感，不要把内容塞满。
- 移动端优先保证单手阅读舒适。
- 同一区块内可以轻微错位，但不要视觉混乱。
- 留白要服务于照片和文字，不要像空洞海报。

## 卡片规范

当前方向是减少“UI 卡片感”，增加“照片、纸张、便签、纪念册”的触感。

可用样式：

- 照片使用白色相纸感容器。
- 生活碎片使用纸张和胶带感。
- 日记、留言、纪念日可以保留轻边框和低透明背景。
- 重要信息可用边线、纸条、轻阴影，而不是厚重圆角卡片。

避免：

- 所有组件都统一大圆角。
- 强玻璃拟态堆叠。
- 过度整齐的网格卡片。
- 只为了装饰而加图标。

当前圆角使用状况：

- 导航和按钮可保留圆角，因为它们是操作控件。
- 内容卡片应逐步减少大圆角，更多使用纸张、相纸、边线。
- 相册照片容器不使用明显圆角，更接近实体照片。

## 图片规范

图片是后续网站的核心。

当前图片来源：

- `public/images/hero-couple.jpg`
- `public/images/story-01.jpg` 到 `story-04.jpg`
- `public/images/photo-01.jpg` 到 `photo-08.jpg`
- `public/images/gate.jpg`

内容配置在 `data/site.ts`：

- Hero 背景：`couple.backgroundImage`
- 故事图片：`story[].image`
- 相册图片：`photos[].src`
- 相册比例：`photos[].ratio`
- 相册空位：`photoPlaceholders`

图片使用原则：

- 优先真实照片。
- 保留自然光、轻微模糊、胶片颗粒。
- 避免插画感和 AI 抽象图形。
- 支持横图、竖图、方图和宽图。
- 照片说明应包含日期、地点和很小的生活细节。

## 动效规范

当前动效库为 Framer Motion。

已使用的动效类型：

- 首屏进入：`opacity + y + blur`
- 背景呼吸：`scale [1, 1.018, 1]`
- 鼠标视差：`useMotionValue + useSpring + useTransform`
- 滚动出现：`whileInView`
- 相册 hover：轻微上移、旋转归正
- 弹窗：淡入和轻 scale
- 播放器：固定底部，操作按钮 transition

参数倾向：

- duration：`0.8s` 到 `1.15s` 用于普通入场。
- Hero 背景呼吸：`24s`。
- live photo 轻动效：`7s`。
- easing：优先 `[0.22, 1, 0.36, 1]` 或 `easeInOut`。

原则：

- 慢、轻、柔和。
- 不大幅弹跳。
- 不做廉价粒子爆炸或爱心特效。
- 动效应像空气、光、翻页，而不是炫技。
- 移动端动效要轻，避免影响滚动。

## 响应式规范

断点策略：

- 默认移动端单列。
- `sm` 增加横向空间和部分双列。
- `lg` 进入桌面分栏和更大留白。
- 相册：移动端 1 列，`sm` 2 列，`lg` 3 列。

移动端重点：

- 顶部导航必须保持可点击，不遮挡首屏标题。
- Hero 标题不应超过屏宽。
- 纸条和卡片文字不应溢出。
- 底部播放器需要留足安全区。
- 主要按钮高度要适合触控。
- 页面不要依赖 hover 才能理解内容。

## 内容规范

所有核心内容应继续集中在 `data/site.ts`。

禁止把以下内容写死在组件里：

- 标题、文案、日期、城市、纪念日。
- 图片路径、音乐路径。
- 相册图片和照片比例。
- 日记默认内容、留言默认内容。
- Footer 和联系方式。

组件可以写死的是：

- 布局逻辑。
- 动效参数。
- Tailwind 样式。
- 功能性 aria label 的引用方式。

文案方向：

- 写真实生活细节，而不是泛泛的浪漫句。
- 避免“记录美好时光”“属于我们的故事”等模板词。
- 多写地点、天气、时间、具体动作。
- 保持中文自然口吻。
