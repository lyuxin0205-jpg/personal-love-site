# PROJECT_CONTEXT

## 网站定位

这是一个私人情侣生活记录网站，不是商业落地页，也不是设计作品集。当前项目围绕“两个人长期认真生活过的痕迹”展开，用来记录恋爱日常、照片、故事、城市、纪念日、愿望、双人日记和留言。

项目当前使用 Next.js App Router、Tailwind CSS v4、Framer Motion、React 19。页面入口在 `app/page.tsx`，内容统一由 `data/site.ts` 管理。

## 设计理念

当前设计方向已经从早期的黑金、商务、模板化视觉，调整为低饱和黄绿蓝的日系生活记录感。

核心气质：

- 像多年后重新打开的私人纪念册。
- 像日系摄影博客、胶片摄影网站、小红书高质量情侣主页。
- 以真实照片、日期、城市、天气、生活片段为核心，而不是以 UI 卡片为核心。
- 保留轻微电影感，但避免夸张玻璃拟态、爱心特效、廉价渐变和抽象插画感。

当前页面强调：

- 奶绿色、青绿色、米白色。
- 夏天、海边、树影、便利店、城市散步、晚风。
- 胶片颗粒、柔和空气感、轻微错位、纸张感和照片墙感。
- 中文优先，避免英文模板文案。

## 当前视觉风格

全局视觉主要定义在 `app/globals.css`：

- 页面背景使用米白到浅绿、浅蓝的纵向渐变。
- `body::before` 提供轻微胶片颗粒。
- `body::after` 提供柔和空气感和弱暗角。
- `.cinema-title` 使用 serif 字体族，营造轻电影感。
- `.glass` 仍保留，但当前方向应谨慎使用，避免过强玻璃拟态。

当前 Hero 在 `components/hero.tsx`：

- 使用 `couple.backgroundImage` 作为首屏背景。
- 背景有极慢缩放和鼠标轻微视差。
- 标题较克制，中文优先。
- 首页生活纸条来自 `couple.heroNotes`。
- 恋爱天数由 `couple.startDate` 计算。

相册在 `components/photo-wall.tsx`：

- 使用 Masonry 风格 columns 布局。
- 每张照片有轻微旋转和错位。
- 支持不同照片比例。
- 支持空位 `photoPlaceholders`，体现相册长期增长。

## 色彩系统

当前 CSS 变量：

- `--ink: #f4f6eb`：页面主背景，米白偏奶绿。
- `--ink-soft: #e5efe6`：更浅的柔和背景。
- `--cream: #315f5a`：当前主要文字色，实际是深青灰/墨绿色。
- `--warm: #6f9f91`：辅助文字色，偏青绿色。
- `--gold: #94bc67`：点缀色，当前更接近黄绿色，不是金色。
- `--rose: #d68f7e`：少量错误/提示用暖色。
- `--glass: rgba(255, 255, 248, 0.68)`：轻玻璃背景。

命名上仍保留早期的 `cream/gold`，但实际视觉已经转为自然黄绿蓝。后续如果重构，可考虑改名为更贴切的 `paper/leaf/moss/sea`，但当前不必为了命名大改页面。

## 动效原则

当前动效来自 Framer Motion 和 CSS transition：

- 页面加载幕布：`LoadingCurtain`。
- 顶部阅读进度：`motion.div` + `useScroll`。
- Hero 背景：慢速缩放，鼠标轻微视差。
- 内容入场：淡入、轻微上移、少量 blur。
- 相册照片：轻微 hover、轻微旋转复位、个别照片 slow breathing。
- 弹窗：照片预览使用淡入和轻 scale。
- 粒子层：`Starfield` 只做非常轻的空气漂浮。

后续动效原则：

- 慢、轻、柔和。
- 不做强弹跳，不做大幅位移，不做炫技转场。
- 动效服务于“回忆被慢慢翻开”，不是展示技术。

当前单页结构在 `app/page.tsx`：

4. 首页 Hero：`Hero`
5. 我们的故事：`StoryTimeline`

8. 3D 翻页相册：`ThreeDAlbum`

11. 城市记录：`LoveMap`
12. 纪念日提醒：`AnniversaryReminders`
13. 愿望清单：`WishList`
14. Footer

## 内容管理方式

当前所有核心内容集中在 `data/site.ts`：

- 情侣姓名、密码、恋爱开始日期、首页背景图。
- 首页标题、副文案、生活纸条。
- 导航、按钮、placeholder、footer。
- 时间轴故事、照片、相册空位。
- 生活碎片、愿望清单、纪念日、城市记录、日记默认内容。
- 音乐标题和音频路径。

后续维护优先只修改 `data/site.ts`，不改组件。

## 移动端规范

当前页面以移动端优先为基础：

- section 使用 `px-5` 起步，桌面逐步提升到 `sm:px-8`、`lg:px-14`。
- Hero 标题使用 `clamp()` 控制尺寸。
- 导航固定在顶部，宽度为 `calc(100%-1.25rem)`，适配窄屏。
- 相册移动端单列，`sm` 进入双列，`lg` 进入三列。
- 日记/留言、纪念日/愿望在桌面分栏，移动端纵向排列。
- 底部播放器固定，页面底部 footer 和 future section 留出较大 `pb`。

后续移动端优化应优先检查：

- 音乐播放器是否遮挡按钮。
- Hero 信息纸条是否过长。
- 相册空位和照片说明是否挤压。
- 长中文标题是否换行自然。

## 后续开发方向

建议后续只做精修，不再大改风格：

- 用真实情侣照片替换 `public/images` 下的占位图。
- 继续减少纯 UI 卡片，强化照片、纸张、日期和地点。
- 增加更真实的季节/天气/城市生活信息。
- 为 `data/site.ts` 保持清晰注释，未来可接 CMS。
- 若接后台，优先把 `contentSource.mode` 从 `local-file` 扩展到接口数据，不要让组件直接关心数据来源。
- 后续新增视频、地图、回忆录、云相册时，应继续沿用“数据驱动组件”的结构。

## 本地 Admin 架构

当前已新增 `/admin` 本地内容管理页面，定位是“以后接真正后台之前的轻量编辑器”。

- 默认内容仍来自 `data/site.ts` 和 `siteContent`。
- 前台组件通过 `lib/content-store.tsx` 读取统一内容对象，而不是直接在组件里写死文案。
- Admin 修改后的内容保存在浏览器 `localStorage`，key 为 `couple-site-content-v1`。
- 已支持编辑：首页姓名、开始日期、标题、文案、背景图、首页生活小纸条、导航、板块标题、故事、故事图片、相册、照片上传、音乐、纪念日、城市记录、日记默认内容和页脚。
- 未来接 CMS 或数据库时，优先替换 `lib/content-store.tsx` 的读取/保存来源，尽量保持 `siteContent` 数据结构不变。

本地版本的上传图片和音乐会以 data URL 存在浏览器里，适合占位和少量测试。真实长期使用时，应把上传文件迁移到对象存储或云端相册，再把 URL 写回同一套内容结构。
